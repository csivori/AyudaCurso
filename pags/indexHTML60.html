Compilado de Instrucciones HTML de Formularios

Los formularios permiten al usuario ingresar información a la aplicación. La etiqueta <form> debe englobar todos los elementos que permiten ingresar diversos tipos de información. Los atributos mas comunes son:

action Especifica la URL o la función JavaScript que debe llamarse al ejecutar el Submit.
method Especifica el metodo de envío de la información:
  • GET: agrega a la URL los campos incluidos en el formulario. Estarían visibles en la URL destino.
  • POST: graba los campos incluidos en el formulario en un espacio especial de almacenamiento del Browser. Serían invisibles al usuario.
target Especifica donde mostrar el HTML resultante de ejecutar action lo muestra en:
  • _blank: una nueva solapa.
  • _self: el mismo recuadro de la solapa.
  • _parent: el recuadro padre.
  • _top: la misma solapa.
autocomplete Especifica si permite autocompletar con información histórica: on / off
novalidate Especifica que no se debe validar la información al ejecutar el Submit

Los elementos que puede contener un formulario son:

<input> Es el campo de ingreso de datos por excelencia. Implementa varios tipos pre-existentes que automatizan el ingreso de información específica. También incluye validaciones básicas de campo. Mas información...
type
Información: text / number / time / date / datetime-local / month / week / tel / color / password / url / email / file / image / range / hidden / checkbox / radio
Botones: button / submit / reset
<textarea> Es similar a un input pero multi-línea. Mediante cols y rows indica su dimensión en pantalla. Mas información...
<label> Mediante el atributo for indica el id de un <input> o <textarea> al que se asocia.
<select> +
<optgroup> +
<option>		 Implementa un objeto ComboBox o DropDown para seleccionar de una lista de posibles valores. Mas información...
<datalist> Define una lista de posibles valores, que en combinación con un input, Implementa un objeto ComboBox o DropDown Editable que permite seleccionar o ingresar un valor que no figura en la lista. Mas información...
<button> A diferencia de un <input type="button">, éste permite estilizar el título e incorporar imágenes. Se debe especificar el type="submit, reset o button" para mejorar el comportamiento del formulario.
<fieldset>
<legend> 		Implementa un objeto GroupBox, envolviendo una serie de elementos en su interior por afinidad semántica (ej: Domicilio, Datos Personales, etc...). Mas información...
<output> Sirve para mostrar el resultado de un proceso. (ej: Resultado de una cálculo matemático) for indica el id de el o los <input> asociados. Mas información...
